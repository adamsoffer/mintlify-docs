This guide takes you from zero to your first inference call on the Livepeer network. You will sign up for the Developer Dashboard, get an API key, install the Python SDK, and run an image-to-image inference job. The whole path takes under five minutes on a fresh environment.

**Prerequisites:**
- Python 3.8 or later
- A Livepeer account (you'll create one in Step 1)

<Tip>
  If you use Cursor or Claude, you can call the Livepeer network directly from your editor via the Livepeer MCP server — no terminal required. See [Use with MCP tools](/guides/mcp-tools) to set that up instead.
</Tip>

<Steps>
  <Step title="Create your account">
    Go to [livepeer.org/dashboard](https://livepeer.org/dashboard) and sign up. No enterprise contract or sales conversation is required — the sign-up process is self-serve.
  </Step>

  <Step title="Get your API key">
    After signing in, navigate to **Settings → API Keys** and create a new key. Copy it — you will use it in the next step.

    Your API key is a secret. Do not commit it to source control. Store it in an environment variable or secrets manager.
  </Step>

  <Step title="Install the Python SDK">
    Install the Livepeer SDK with pip:

    ```bash
    pip install livepeer
    ```
  </Step>

  <Step title="Make your first inference call">
    The example below runs an image-to-image pipeline. Replace `"your-api-key"` with the key you copied in Step 2.

    <CodeGroup>

    ```bash Install
    pip install livepeer
    ```

    ```python First call
    import livepeer

    client = livepeer.Livepeer(api_key="your-api-key")

    response = client.inference.image_to_image(
        prompt="anime style, vibrant colors",
        image_url="https://example.com/input.jpg"
    )

    print(response.image_url)
    ```

    </CodeGroup>

    The SDK authenticates your request, routes it to an available orchestrator on the network, and returns a URL pointing to the transformed image.
  </Step>

  <Step title="View the result">
    Open the URL printed by `response.image_url` in your browser to see the output. The image was generated by an independent GPU provider on the Livepeer network.

    From here you can explore other inference pipelines in the [API reference](/api-reference/overview), or work through the [detailed first-call guide](/guides/first-inference-call) for more options and parameters.
  </Step>
</Steps>

<Note>
  The Livepeer network operates at an estimated 90–95% uptime. Jobs that fail mid-stream are not charged. If a job fails, the error includes orchestrator attribution so you can diagnose what happened. The network is designed for builders who trade enterprise-grade SLAs for dramatically lower cost and open access.
</Note>
